
Generate a Modelica block named `ComplexToReal`, which converts a complex input signal into its Cartesian components (real and imaginary parts). The model should extend from `Modelica.ComplexBlocks.Icons.ComplexBlock`. The model should include the following: 1. **Functionality**: The model takes a complex input and outputs its real and imaginary components. It should also handle the option to process the conjugate of the input. 2. **Inputs and Outputs**: The model should have one complex input `u` of type `Modelica.ComplexBlocks.Interfaces.ComplexInput`, and two real outputs `re` and `im` of type `Modelica.Blocks.Interfaces.RealOutput`. 3. **Parameters**: Include a Boolean parameter `useConjugateInput` with a default value of `false`, which determines if the input should be treated as its conjugate. 4. **Equations**: The real part 're' of the input 'u' should be assigned to `re`, and the imaginary part 'im' to `im`. If `useConjugateInput` is `true`, negate the imaginary part before assigning it to `im`.

//modelica
```block ComplexToReal extends Modelica.ComplexBlocks.Icons.ComplexBlock
  parameter Boolean useConjugateInput=false "If true, input is treated as its conjugate";
  Modelica.ComplexBlocks.Interfaces.ComplexInput u "Complex input";
  Modelica.Blocks.Interfaces.RealOutput re "Real part of input";
  Modelica.Blocks.Interfaces.RealOutput im "Imaginary part of input";
equation
  re = if useConjugateInput then conjugate(u.re) else u.re;
  im = if useConjugateInput then -conjugate(u.im) else u.im;
end ComplexToReal;
```

This block can be used to separate complex signals into their real and imaginary components, which is useful in many signal processing applications.
