
We have a container, and we want to keep the height of the water (`waterLevel`) in the container at a target height (`target_height`) of 1 meter. The water height (`waterLevel`) in the pool starts at 0.2m. Use proportional differential control to implement the above control system (defining the height error as `error`, and splitting the logic into proportional control `Pcontrol` and differential control `Dcontrol`) and simulate the change of the water level in the pool over time, where the proportional coefficient (`Kp`) is 1 and the differential coefficient (`Kd`) is 0.1. Please help me implement a modelica model named `PDController`.

//modelica
```model PDController
  parameter Real target_height = 1 "Target height of water";
  parameter Real Kp = 1 "Proportional coefficient";
  parameter Real Kd = 0.1 "Differential coefficient";
  parameter Real waterLevel(start = 0.2) "Current water level in the container";
  parameter Real error(start = 0.8) "Height error";
  parameter Real Pcontrol(start = 0.8) "Proportional control";
  parameter Real Dcontrol(start = 0) "Differential control";
  parameter Real previous_error(start = 0.8) "Previous error";
  parameter Real dt = 0.01 "Time step";
equation
  // Proportional control
  Pcontrol = Kp * error;
  // Differential control
  Dcontrol = Kd * (error - previous_error) / dt;
  // Update water level
  der(waterLevel) = Pcontrol + Dcontrol;
  // Update error
  error = target_height - waterLevel;
  // Update previous error
  previous_error = error;
end PDController;
```

This model is a simple model of a PD controller. The proportional control `Pcontrol` is calculated as the proportional coefficient `Kp` times the error `error`. The differential control `Dcontrol` is calculated as the differential coefficient `Kd` times the change in error over time `dt`. The change in the water level `waterLevel` is then calculated as the sum of the proportional and differential control. The error is updated to be the difference between the target height and the current water level. The previous error is also updated to be the current error.

Please note that this model does not include any saturation or anti-windup logic, which would be necessary in a real-world application to prevent the controller from outputting values that would cause the system to behave in an undesirable way.

Also, the model assumes that the time step `dt` is constant and known in advance. In a real-world application, the time step would typically be determined automatically by the simulation software.

Finally, the model assumes that the proportional and differential coefficients `Kp` and `Kd` are constant. In a real-world application, these coefficients might need to be adjusted over time based on feedback from the system.
