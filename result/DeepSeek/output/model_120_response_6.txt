
Generate a Modelica model named `PID_Controller` for a PID (Proportional-Integral-Derivative) Controller subsystem. The model should:

1. Extends the `ModelicaByExample.Architectures.SensorComparison.Interfaces.Controller`.
2. The model includes the following parameters :
 - Real 'k' is Gain of controller;
 -Modelica.SIunits.Time 'Ti' is Time constant of Integrator block;
 - Modelica.SIunits.Time 'Td' isTime constant of Derivative block;
 - Real 'yMax' is Upper limit of output.
3. Utilize a 'Modelica.Blocks.Continuous.LimPID' block named 'PID' in the protected section, configured with specific parameters for controller gain (k), time constant of the Integrator block (Ti), time constant of the Derivative block (Td), and an upper limit of output (yMax).
4. Establish the following signal connections in the equation section: The reference tracking connects the external setpoint to the PID block's setpoint input (PID.u_s).
The feedback loop links the measured signal to the PID block's measurement input (PID.u_m).
The control action is routed from the PID block's output (PID.y) to the external command interface.

//modelica
```model PID_Controller
  extends ModelicaByExample.Architectures.SensorComparison.Interfaces.Controller;
  parameter Modelica.SIunits.Time Ti(start=1);
  parameter Modelica.SIunits.Time Td(start=0.1);
  parameter Real k(start=1);
  parameter Real yMax(start=100);
  protected 
  Modelica.Blocks.Continuous.LimPID PID;
  equation 
  PID.k = k;
  PID.Ti = Ti;
  PID.Td = Td;
  PID.yMax = yMax;
  PID.u_s = reference;
  PID.u_m = measured;
  command = PID.y;
end PID_Controller;
```
