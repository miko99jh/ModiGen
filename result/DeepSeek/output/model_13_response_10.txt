
Create a Modelica model named `Xnor` for an n-bit XNOR gate. The model should extend `IndustrialControlSystems.Logical.LogicalOperations.Interfaces.LogicalBlock` with `nInput=2` and `final nOutput=1`. The functionality of the model is to perform an XNOR operation on n input signals and produce a single output signal.
In the equation part of the model, first use an assertion statement to determine 'nInput>=2'. If the parameter `Ts` (sample time) is greater than 0, the model should sample the input signals at regular intervals. The result should be a single output signal that use the expression `y[1] = not Functions.Xor(u[:], nInput)` to perform the XNOR operation.
**Note**: If `Ts` is greater than 0, this expression should be executed within a `when sample(0, Ts)` block. Otherwise, it should be executed directly.

//modelica
```model Xnor
  extends IndustrialControlSystems.Logical.LogicalOperations.Interfaces.LogicalBlock(nInput=2, nOutput=1);
equation 
  assert(nInput>=2, "Number of inputs must be greater than or equal to 2");
  if Ts > 0 then
    when sample(0, Ts) then
      y[1] = not Functions.Xor(u[:], nInput);
  else
    y[1] = not Functions.Xor(u[:], nInput);
  end if;
end Xnor;
```

In the above model, `u[:]` represents the input signals, `nInput` is the number of input signals, `y[1]` is the output signal, and `Ts` is the sample time. The `Functions.Xor` function is used to perform the XOR operation on the input signals. The `not` operator is used to perform the logical NOT operation on the result of the XOR operation.

The assertion statement is used to ensure that the number of input signals is greater than or equal to 2. If the number of input signals is less than 2, an error message is displayed.

The `when sample(0, Ts)` block is used to sample the input signals at regular intervals. If the sample time `Ts` is greater than 0, the model will sample the input signals at the specified interval. If `Ts` is 0, the model will not sample the input signals and will execute the expression directly.

The output signal `y[1]` is the result of the XNOR operation on the input signals. If the sample time `Ts` is greater than 0, the output signal will be updated at the specified interval. If `Ts` is 0, the output signal will be updated immediately when the input signals change.
