```modelica
block FractionalDelay
  extends Modelica.Clocked.IntegerSignals.Interfaces.PartialClockedSISO;

  parameter Integer shift(min=0) = 0 "Delay as a product of the sample period and the ratio shift/resolution";
  parameter Integer resolution(min=1) = 1 "Time quantization resolution of the sample period";

protected
  Integer n = div(shift, resolution) "Number of full sample periods to delay";
  Integer u_buffer[n+1](each start=0) "Buffer to store previous values of the input signal";
  Boolean first(start=true) "Flag to identify the first clock tick";

equation
  // Update the flag to indicate the first tick has passed
  first = false;

  // Buffer management
  if previous(first) then
    // Initialize buffer with the current input on the first tick
    for i in 1:n+1 loop
      u_buffer[i] = u;
    end for;
  else
    // Shift buffer: prepend current input and drop the oldest value
    u_buffer[1] = u;
    for i in 2:n+1 loop
      u_buffer[i] = previous(u_buffer[i-1]);
    end for;
  end if;

  // Output calculation using shiftSample operator
  y = shiftSample(u_buffer[n+1], shift, resolution);

end FractionalDelay;
```

This Modelica block `FractionalDelay` is designed to delay an incoming clocked input signal by a fractional multiple of the sample period. It uses a buffer to store previous input values and calculates the output based on the specified delay parameters. The model is suitable for applications requiring precise control over signal timing, such as digital communication systems.
