To create a Modelica model named `ShiftSample` that demonstrates the usage of a `ShiftSample` block for clock signals, you can follow the structure below. This model includes the specified components and establishes the necessary connections between them.

```modelica
model ShiftSample
  // Import necessary Modelica libraries
  import Modelica.Clocked.ClockSignals.Clocks.PeriodicExactClock;
  import Modelica.Clocked.ClockSignals.Sampler.ShiftSample;
  import Modelica.Clocked.RealSignals.Sampler.SampleClocked;
  import Modelica.Blocks.Sources.Sine;
  import Modelica.Clocked.Types.Resolution;

  // Components
  PeriodicExactClock periodicClock(factor=20, resolution=Resolution.ms) "Periodic clock with 20 ms period";
  ShiftSample shiftSample1(shiftCounter=4, resolution=3) "ShiftSample block with shiftCounter=4 and resolution=3";
  SampleClocked sample1 "SampleClocked block to sample the sine wave";
  Sine sine1(frequency=2, offset=0.1, startTime=0) "Sine wave source with frequency=2 Hz, offset=0.1";

equation
  // Connect the periodic clock to the ShiftSample block
  connect(periodicClock.y, shiftSample1.u);

  // Use the shifted clock signal to drive the SampleClocked block
  connect(shiftSample1.y, sample1.clock);

  // Sample the continuous output of the sine wave using the SampleClocked block
  connect(sine1.y, sample1.u);

  // Output the sampled signal (optional, for visualization or further processing)
  // RealOutput sampledSignal;
  // connect(sample1.y, sampledSignal);

end ShiftSample;
```

### Explanation:
- **PeriodicExactClock**: Generates a periodic clock signal with a period determined by the `factor` and `resolution`. Here, it is set to 20 ms.
- **ShiftSample**: Shifts the input clock signal by a specified number of clock ticks (`shiftCounter`) and uses a specified resolution.
- **SampleClocked**: Samples a continuous-time signal based on the clock signal provided to it.
- **Sine**: Generates a continuous sine wave signal with a specified frequency, offset, and start time.

### Connectivity:
1. The `periodicClock` generates a clock signal that is fed into the `shiftSample1` block.
2. The `shiftSample1` block shifts the clock signal and outputs it to the `sample1` block.
3. The `sample1` block uses the shifted clock signal to sample the output of the `sine1` block, creating a discrete-time representation of the sine wave.

This model demonstrates how to use clock signals and the `ShiftSample` block to manipulate and sample continuous signals in a discrete-time manner.
