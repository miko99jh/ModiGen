To create a Modelica block named `SignalRanker` that ranks output signals in descending order, you can extend the `Modelica.Blocks.Interfaces.MIMO` interface. This block will use the `Modelica.Math.Vectors.sort` function to sort the input vector. Below is the implementation:

```modelica
block SignalRanker
  extends Modelica.Blocks.Interfaces.MIMO;

protected 
  Real sortedU[size(u, 1)];

equation 
  // Ensure the number of outputs matches the number of inputs
  assert(nout == nin, "Number of outputs must match number of inputs.");

  // Sort the input vector in descending order
  sortedU = Modelica.Math.Vectors.sort(u, ascending=false);

  // Assign the sorted values to the output
  y = sortedU;

end SignalRanker;
```

### Explanation:
- **Extending MIMO**: The block extends `Modelica.Blocks.Interfaces.MIMO`, which is a multi-input, multi-output interface. This ensures that the block can handle multiple input and output signals.
- **Protected Variable**: `sortedU` is a protected variable used to store the sorted version of the input vector `u`.
- **Equation Section**: 
  - An `assert` statement ensures that the number of outputs (`nout`) matches the number of inputs (`nin`).
  - The `Modelica.Math.Vectors.sort` function is used to sort the input vector `u` in descending order by setting `ascending=false`.
  - The sorted vector `sortedU` is then assigned to the output `y`.

This block will take an input vector `u`, sort it in descending order, and output the sorted vector `y`.
