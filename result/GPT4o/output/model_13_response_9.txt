To create a Modelica model named `Xnor` for an n-bit XNOR gate, you can follow the structure below. This model extends `IndustrialControlSystems.Logical.LogicalOperations.Interfaces.LogicalBlock` and implements the required functionality. The model includes an assertion to ensure `nInput >= 2` and handles the sampling of input signals based on the `Ts` parameter.

```modelica
model Xnor
  extends IndustrialControlSystems.Logical.LogicalOperations.Interfaces.LogicalBlock(
    nInput=2, final nOutput=1);

  parameter Real Ts = 0 "Sample time";

equation
  // Assert that the number of inputs is at least 2
  assert(nInput >= 2, "The number of inputs must be at least 2.");

  if Ts > 0 then
    // When Ts is greater than 0, use a when sample block
    when sample(0, Ts) then
      y[1] = not Functions.Xor(u[:], nInput);
    end when;
  else
    // Directly execute the XNOR operation
    y[1] = not Functions.Xor(u[:], nInput);
  end if;
end Xnor;
```

### Explanation:

- **Model Extension**: The model extends `IndustrialControlSystems.Logical.LogicalOperations.Interfaces.LogicalBlock` with `nInput=2` and `final nOutput=1`.

- **Parameters**: 
  - `Ts`: A parameter representing the sample time. If `Ts` is greater than 0, the model samples the input signals at regular intervals.

- **Assertions**: 
  - An assertion is used to ensure that `nInput` is at least 2.

- **Equation Section**:
  - If `Ts` is greater than 0, the XNOR operation is performed within a `when sample(0, Ts)` block to sample the input signals at regular intervals.
  - If `Ts` is 0 or less, the XNOR operation is performed directly without sampling.

- **XNOR Operation**: 
  - The expression `y[1] = not Functions.Xor(u[:], nInput)` is used to perform the XNOR operation on the input signals.

This model should meet the requirements for an n-bit XNOR gate with the specified behavior.
