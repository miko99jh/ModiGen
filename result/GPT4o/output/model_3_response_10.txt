To create a Modelica block named `SignalRanker` that ranks output signals in descending order, you can extend the `Modelica.Blocks.Interfaces.MIMO` interface. This block will use the `Modelica.Math.Vectors.sort` function to sort the input vector. Here's how you can implement this:

```modelica
block SignalRanker
  extends Modelica.Blocks.Interfaces.MIMO;

  // Ensure the number of outputs matches the number of inputs
  parameter Integer nin = 1 "Number of inputs and outputs";
  parameter Integer nout = nin "Number of outputs";

  // Input and output connectors
  input Real u[nin] "Input vector";
  output Real y[nout] "Output vector";

protected 
  Real sortedU[nin];

equation 
  // Sort the input vector in descending order
  sortedU = Modelica.Math.Vectors.sort(u, ascending=false);

  // Assign sorted values to output
  for i in 1:nout loop
    y[i] = sortedU[i];
  end for;

end SignalRanker;
```

### Explanation:
- **Extending MIMO**: The block extends `Modelica.Blocks.Interfaces.MIMO`, which is suitable for blocks with multiple inputs and outputs.
- **Parameters**: `nin` and `nout` are parameters that define the number of inputs and outputs, respectively. They are set to be equal to ensure the number of outputs matches the number of inputs.
- **Connectors**: The block has input and output connectors `u` and `y`, both of which are vectors of size `nin` and `nout`.
- **Protected Variable**: `sortedU` is a protected variable used to store the sorted input vector.
- **Equation Section**: The `Modelica.Math.Vectors.sort` function is used to sort the input vector `u` in descending order. The sorted values are then assigned to the output vector `y`.
